9:T954,<h1>자료실 사용법</h1>
<p>이 자료실은 <strong>사람이 읽고 보는 외부 학습 자료</strong>를 모아두는 곳입니다. Skills 섹션이 <em>Claude Code 같은 에이전트가 직접 돌리는</em> 스킬이라면, Resources는 우리(사람) 머리에 들어가야 할 자료입니다.</p>
<h2>글 1개에 들어갈 수 있는 것</h2>
<ul>
<li><strong>외부 링크</strong> (<code>source_url</code>) — Coursera 강의, YouTube 영상, 논문 PDF, 블로그 글 등</li>
<li><strong>첨부파일</strong> (<code>attachments</code>) — 슬라이드 PDF, 예제 코드 zip, 강의 노트 등 우리 repo에 직접 올린 것</li>
</ul>
<p>둘 다 옵션입니다. 외부 링크만 있어도 되고, 첨부만 있어도 되고, 본문만 있어도 됩니다.</p>
<h2>새 자료 올리는 법</h2>
<ol>
<li><code>content/resources/</code> 안에 <code>YYYY-MM-DD-제목.md</code> 형식으로 파일 생성</li>
<li>frontmatter에 메타 채우기 (이 글의 frontmatter 복사해서 수정하면 됨)</li>
<li>첨부가 있으면 <code>public/resources/files/</code> 안에 파일 두고, frontmatter <code>attachments:</code> 배열에 파일명 적기</li>
<li>본문에 <em>왜 이걸 봐야 하는지</em> + <em>어디부터 보면 되는지</em> 짧게 적기</li>
<li>push → GitHub Actions가 빌드 → <code>lab.haiinu.com/resources/</code> 에 자동 반영</li>
</ol>
<h2>첨부 파일 크기 주의</h2>
<p>GitHub repo는 단일 파일 100MB 한계가 있고, 권장은 50MB 이하입니다. 큰 PDF나 슬라이드 묶음은 Google Drive에 올리고 <code>source_url</code>로 링크 거는 게 안전합니다.</p>
<h2>본문에 무엇을 적을지</h2>
<p>학생이 카드 보고 <em>볼지 말지</em> 30초 안에 판단할 수 있게:</p>
<ul>
<li><strong>한 줄 요약</strong> — 무엇에 대한 자료인지</li>
<li><strong>추천 이유</strong> — 왜 이걸 봐야 하는지 (다른 비슷한 자료와의 차이)</li>
<li><strong>시작점</strong> — 처음부터 다 볼 필요 없으면 어디서 시작할지</li>
<li><strong>함정</strong> — 빠지기 쉬운 오해나 주의점이 있으면</li>
</ul>
<p>길게 쓸 필요 없습니다. 다 봤으면 이 글처럼 5문단 이내가 적당.</p>
