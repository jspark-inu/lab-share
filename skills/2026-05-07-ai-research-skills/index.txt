9:T136a,<h1>AI Research SKILLs</h1>
<blockquote>
<p><strong>한 줄</strong>: Orchestra Research가 만든 <strong>98개 AI 연구 자동화 스킬</strong> 라이브러리. literature survey → ideation → 모델 학습 → 평가 → 논문 작성까지 <em>연구 라이프사이클 전체</em>를 모듈로 쪼개놨다.</p>
</blockquote>
<ul>
<li><strong>Repo</strong>: <a href="https://github.com/Orchestra-Research/AI-Research-SKILLs">Orchestra-Research/AI-Research-SKILLs</a></li>
<li><strong>라이선스</strong>: MIT</li>
<li><strong>설치</strong>: <code>npx @orchestra-research/ai-research-skills</code> (인터랙티브) — <code>~/.orchestra/skills/</code>에 설치되고 각 에이전트(Claude Code, Hermes, OpenCode, Cursor, Gemini CLI 등)에 심볼릭 링크.</li>
</ul>
<h2>왜 이걸 쓰나</h2>
<p>AI 연구를 자동화하려면 한 도메인만 깊으면 안 된다. <strong>Megatron-LM 분산 학습</strong>도 알아야 하고, <strong>vLLM 추론</strong>도 알아야 하고, <strong>TRL post-training</strong>도 알아야 하고, <strong>mech interp</strong>도 알아야 한다. 한 사람이 다 외우는 건 불가능.</p>
<p>그래서 Orchestra는 도메인을 23개 카테고리로 나누고, 각 카테고리당 <em>공식 레포 + 실제 GitHub 이슈 + 프로덕션 워크플로우</em>를 반영한 스킬을 만들었다. 에이전트가 작업 맥락에 맞는 스킬을 자동 라우팅한다(두-루프 아키텍처: <code>autoresearch</code>가 오케스트레이션, 도메인 스킬이 실행).</p>

<h2>우리 랩에서 차용한 부분</h2>
<p>전부 다 깔지 않아도 <em>부분 차용</em>이 가능하다. 우리는 <strong><code>research-brainstorm</code></strong> 스킬만 가져다가 <code>~/.claude/skills/</code>에 두고 쓴다.</p>
<ul>
<li><strong>research-brainstorm</strong> — 가설 생성용 <em>18개 인지 프레임워크</em>
<ul>
<li>10개 Workflow Framework: 문제-해법 판별, Abstraction Ladder, Tension Hunting, Cross-Pollination, What Changed, Failure Analysis, Simplicity Test, Stakeholder Rotation, Composition/Decomposition, Explain-It Test</li>
<li>8개 Cognitive Engine: Bisociation, Problem Reformulation, Analogical Reasoning, Constraint Manipulation, Negation/Inversion, Abstraction Laddering, Adjacent Possible, Janusian Thinking</li>
<li><strong>사회과학·경영학 연구에도 그대로 매핑 가능</strong> — 우리 랩이 GABM(Generative Agent-Based Modeling) 가설 다각도 생성에 쓰고 있음</li>
</ul>
</li>
</ul>
<h2>시작하는 법 (가장 작은 예시)</h2>
<pre><code># 학생: 모호한 연구 아이디어가 있을 때
"learning ABM 환경에서 학생들의 협력 패턴이 LLM persona 변화로 어떻게 영향받나"

# Skill 호출
/research-brainstorm

# 18개 프레임워크 중 3-5개 자동 선택
# 각 프레임워크 렌즈로 가설 생성:
# - Tension Hunting → "협력 vs 무임승차 긴장이 personality trait와 어떻게 상호작용?"
# - Negation/Inversion → "LLM이 *비협력적*일 때 학생 협력은 오히려 늘어날까?"
# - Adjacent Possible → "ABM 외에 RL multi-agent에서 비슷한 현상 보고됐나?"
</code></pre>
<h2>함정</h2>
<ul>
<li>98개 스킬을 다 깔면 토큰·디스크·인지부담이 큼. <strong>필요한 것만 부분 차용</strong>이 권장.</li>
<li>AI 연구용으로 만들어졌지만 사회과학 연구에도 <em>구조</em> 자체는 적용된다 — 다만 도메인 용어를 머리속에서 매핑해야 함 (예: "ML 모델" → "에이전트 페르소나").</li>
<li><code>autoresearch</code> 오케스트레이션 레이어는 우리는 안 씀 — 우리 랩의 1A6/1A7 같은 자체 연구 파이프라인이 더 좁고 깊음.</li>
</ul>
