# 킥애스 포트폴리오 프로젝트 만들기

Chapter 2. "그냥 한 번 해본 프로젝트"가 아니라 면접관이 *이거 더 듣고 싶다*고 말하게 만드는 포트폴리오를 만드는 법.

## 다루는 것

- 어떤 종류의 프로젝트가 통하는가 (Kaggle 따라하기 vs 자체 데이터)
- 임팩트를 보여주는 프레이밍
- GitHub README와 블로그 글의 역할 분담

## 누가 봐야 하는가

- 이력서에 쓸 프로젝트가 1-2개뿐인 학생
- "캐글 메달 외에 뭘 더 만들지" 고민 중인 학생
- 포트폴리오는 있는데 면접관이 안 물어보는 학생

## 핵심 메시지 미리보기

기술 스택보다 *어떤 의사결정을 내렸고 왜 그랬는가*가 핵심. 1슬라이드 임팩트 요약 → 1문단 문제정의 → 코드/노트북 순서로 깊이가 보이게.
