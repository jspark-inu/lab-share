# 자료실 사용법

이 자료실은 **사람이 읽고 보는 외부 학습 자료**를 모아두는 곳입니다. Skills 섹션이 *Claude Code 같은 에이전트가 직접 돌리는* 스킬이라면, Resources는 우리(사람) 머리에 들어가야 할 자료입니다.

## 글 1개에 들어갈 수 있는 것

- **외부 링크** (`source_url`) — Coursera 강의, YouTube 영상, 논문 PDF, 블로그 글 등
- **첨부파일** (`attachments`) — 슬라이드 PDF, 예제 코드 zip, 강의 노트 등 우리 repo에 직접 올린 것

둘 다 옵션입니다. 외부 링크만 있어도 되고, 첨부만 있어도 되고, 본문만 있어도 됩니다.

## 새 자료 올리는 법

1. `content/resources/` 안에 `YYYY-MM-DD-제목.md` 형식으로 파일 생성
2. frontmatter에 메타 채우기 (이 글의 frontmatter 복사해서 수정하면 됨)
3. 첨부가 있으면 `public/resources/files/` 안에 파일 두고, frontmatter `attachments:` 배열에 파일명 적기
4. 본문에 *왜 이걸 봐야 하는지* + *어디부터 보면 되는지* 짧게 적기
5. push → GitHub Actions가 빌드 → `lab.haiinu.com/resources/` 에 자동 반영

## 첨부 파일 크기 주의

GitHub repo는 단일 파일 100MB 한계가 있고, 권장은 50MB 이하입니다. 큰 PDF나 슬라이드 묶음은 Google Drive에 올리고 `source_url`로 링크 거는 게 안전합니다.

## 본문에 무엇을 적을지

학생이 카드 보고 *볼지 말지* 30초 안에 판단할 수 있게:

- **한 줄 요약** — 무엇에 대한 자료인지
- **추천 이유** — 왜 이걸 봐야 하는지 (다른 비슷한 자료와의 차이)
- **시작점** — 처음부터 다 볼 필요 없으면 어디서 시작할지
- **함정** — 빠지기 쉬운 오해나 주의점이 있으면

길게 쓸 필요 없습니다. 다 봤으면 이 글처럼 5문단 이내가 적당.
