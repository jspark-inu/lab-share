# Ch 11. Case Studies — 실전 케이스 스터디

*Ace the Data Science Interview* Chapter 11. 앞의 1-10장에서 익힌 모든 도구를 *하나의 비즈니스 문제 풀이*에 통합 적용하는 capstone 챕터.

## 다루는 것

- End-to-End 문제 해결 흐름: 문제정의 → 데이터 → 분석 → 모델 → 검증 → 의사결정
- 한 케이스에서 통계·SQL·ML·product sense 어떻게 *섞어 쓰는지*
- 면접관이 *케이스 도중에* 던지는 follow-up 질문 대처법
- "정답이 없는 문제"에서 *내 가정*을 명시적으로 말하는 법

## 누가 봐야 하는가

- 1-10장 다 본 학생 — 마지막 통합 학습용
- 시니어·미드레벨 데이터 직무 면접 (45-60분 케이스 면접 있는 경우)
- 컨설팅 펌 데이터 직무 준비하는 학생

## 핵심 메시지 미리보기

케이스는 답을 *맞추는* 게 아니라 *어떻게 사고하는지*를 보여주는 무대. 가정을 말로 명시하고, 우선순위 트레이드오프를 설명하고, "이 가정이 틀리면 결론이 어떻게 바뀌는지"까지 가는 후보자가 통과한다.
