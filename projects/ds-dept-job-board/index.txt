b:T1571,<h1>데이터과학과 인력사무소</h1>
<blockquote>
<p><strong>한 줄</strong>: 교수·행정 선생님의 <em>실제 문제</em>를 학과 학생이 <em>풀어보는</em> 자체 공모전 매칭 플랫폼.</p>
</blockquote>
<h2>동기 — 두 가지 막힘이 서로를 풀 수 있다</h2>
<p><strong>학생 쪽 막힘</strong></p>
<ul>
<li>"데이터를 분석하고 싶은데 <em>진짜 문제</em>가 안 보인다."</li>
<li>캐글 같은 외부 데이터셋은 <em>문제 정의</em>가 이미 끝난 상태라 학습 효과가 제한적.</li>
<li>포트폴리오에 "내가 직접 발굴한 문제" 항목이 비어 있음.</li>
</ul>
<p><strong>교수·행정 쪽 막힘</strong></p>
<ul>
<li>"개선하고 싶은 게 있는데 <em>직접 들여다볼 시간</em>이 없다."</li>
<li>학과 운영 중 발견한 의문(예: 학생 수강 패턴, 졸업 후 진로 추적, 행정 처리 병목)들이 <em>기록도 안 된 채</em> 사라짐.</li>
</ul>
<p>이 두 막힘은 <em>서로의 답</em>이다. 다만 매칭하는 <em>자리</em>가 없다.</p>
<h2>무엇을 만들 것인가 (예비 그림)</h2>

</tr>

</tr>

<h2>결정 필요 — 오픈 질문 3가지</h2>
<h3>1. 구현 형태 — 학과 사람들이 <em>많이 이용하게</em> 하려면</h3>
<table>
<thead>
<tr>
<th>옵션</th>
<th>장점</th>
<th>약점</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>A. 별도 웹 플랫폼 신규</strong></td>
<td>자유도 ↑, 학과 브랜드화</td>
<td>개발 시간 ↑, 운영 부담</td>
</tr>
<tr>
<td><strong>B. lab-share에 새 카테고리 추가</strong></td>
<td>즉시 가능, 디자인 일관성, 자동 배포 인프라 그대로</td>
<td>lab-share가 <em>연구실용</em>이라 학과 전체에 노출 미묘</td>
</tr>
<tr>
<td><strong>C. 노션 데이터베이스</strong></td>
<td>5분 셋업, 모든 사람이 익숙</td>
<td>노션 계정 의존, 검색·통계 약함</td>
</tr>
<tr>
<td><strong>D. 슬랙 채널 + 스프레드시트</strong></td>
<td>가장 가벼움, 즉시 대화 가능</td>
<td>의뢰가 <em>흘러가버림</em>, 검색·아카이브 약함</td>
</tr>
</tbody>
</table>
<p><strong>현재 직관</strong>: B 또는 C가 빠른 시작. A는 사용자가 검증된 후.</p>
<h3>2. 인센티브 구조 — <em>지속가능</em>하려면</h3>
<p>핵심은 <em>양쪽 모두에게 의미 있는 보상</em>. 한쪽만 매력적이면 의뢰가 쌓이거나 풀이가 쌓이는 <em>비대칭 큐</em>가 됨.</p>
<p><strong>의뢰자(교수·행정) 측</strong></p>
<ul>
<li>의뢰만 등록해도 <em>결과를 받을 가능성</em> 자체가 보상 (현재는 0)</li>
<li>학생 풀이를 <em>데이터과학과 결과물</em>로 학과 홍보·연차 보고서에 활용 가능</li>
<li>(미정) 의뢰 빈도 ↑ 시 학과 차원 인정</li>
</ul>
<p><strong>풀이자(학생) 측</strong></p>
<ul>
<li>포트폴리오 1개 추가 (실제 학과 문제 + 발표 기록)</li>
<li>(미정) 학점 연동 — 캡스톤·졸업과제 대체 인정?</li>
<li>(미정) 우수작 → 장학금 / 채용 추천서 / 컨퍼런스 발표 기회</li>
<li>(미정) 학생-학생 <em>팀 결성</em>도 가능하게?</li>
</ul>
<h3>3. 첫 문제 — 어떤 <em>시드 의뢰</em>로 시작할 것인가</h3>
<p>학생들이 "이런 종류의 문제가 올라오는구나"를 첫 인상으로 갖게 하는 1-2개. 너무 어려우면 진입 장벽, 너무 쉬우면 가치 의심.</p>
<p><strong>후보</strong> (아이디어 단계, 추가·교체 환영):</p>
<ul>
<li>학과 학생 진로 패턴 분석 (졸업 후 5년 추적)</li>
<li>수강신청 데이터 → 트랙 제안 알고리즘</li>
<li>행정 문서 분류 자동화 (조교 시간 절감)</li>
</ul>
<h2>완료 기준 (예비)</h2>
<ul>
<li>첫 학기 안에:
<ul>
<li>의뢰 <strong>5건</strong> 이상 등록</li>
<li>학생 풀이 <strong>3건</strong> 이상 제출</li>
<li>결과 발표회 <strong>1회</strong> 개최</li>
</ul>
</li>
<li>두 번째 학기에:
<ul>
<li>학생 <em>반복 참여율</em> > 30% (한 번 푼 학생이 다음 학기에 다시 참여)</li>
<li>의뢰자 <em>반복 등록률</em> > 50%</li>
</ul>
</li>
</ul>
