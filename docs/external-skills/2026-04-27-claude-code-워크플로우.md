# Claude Code — 학부연구생용 핵심 워크플로우 4개

> Cursor/Copilot 와 다른 점: *프로젝트 전체* 를 보고 *여러 파일* 을 한 번에 다룸.

## 1. 코드베이스 탐색

```
이 repo 전체 구조 설명해줘. 핵심 파일 5개만 골라서.
```

→ 첫 입사 첫 주에 가장 가치 있는 사용법. README보다 빠름.

## 2. 논문 노트 → 실험 코드

```
이 논문 노트(papers/2026-03-26-Uplift-Modeling-AHD.md) 읽고
재현 가능한 최소 실험 1개 설계해줘. mesa 사용.
```

→ 논문 한 편 읽고 *손* 으로 옮기는 시간을 90% 줄임.

## 3. 디버깅

```
이 에러 로그 읽고 root cause 찾아줘.
원인 한 줄 + 수정 1개 + 회귀 방지 테스트 1개.
```

→ "어디 보세요" 가 아니라 *증거 + 단일 수정* 을 요구. 에이전트가 추측 코드 던지는 패턴 차단.

## 4. 일주일 회고

```
git log --since="7 days ago" 분석해서
이번 주 한 일 + 핵심 결정 + 다음 주 할 일 1page 요약.
```

→ 매주 금요일 30분, 학부연구생 주간 회의 자료 자동 생성.

## 함정

- **너무 큰 작업 한 번에 던지지 마라** — 3단계 이상은 계획부터 짜라고 시킴.
- **결과 검증 없이 신뢰하지 마라** — "테스트 돌려서 PASS 보여줘" 까지 요구.
- **세션 격리** — 작업마다 새 세션. 컨텍스트 누수 방지.
